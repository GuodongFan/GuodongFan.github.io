Brainary 不是“关于我”，而是一个持续生长的结构：把想法变成可复用的系统，把实验变成可验证的结果。

Modules

入口即目录。每个模块都可以先是空的，但结构先立住 —— 之后慢慢填。

Notes
短记录 / 灵感 / 读书 / 观察。像日志，但可检索。
Systems
方法论 / 框架 / checklist。可复用的脑回路。
Experiments
AI / code 实验仓。每个实验都有假设、过程、结论。
  • Now building 从零开始：先把“目录结构 + 内容规范”跑起来。
    view
  • Principle 结构优先，结果可证：每条内容都能追溯、复现、迭代。
    manifest

Manifest

Brainary 的三条规则：

  • Make it structured 先给想法一个容器：命名、分类、索引。
  • Make it testable 实验要有输入/输出/指标；记录要能复盘。
  • Make it compounding 让内容互相引用，形成增长的网络。
bootstrap
$ brainary init
→ create: /notes /systems /experiments /archive
→ add:    index.md template + tagging rules
→ deploy: github pages + CNAME brainary.cn

$ brainary add "First experiment: prompt-evals"
→ logged. linked. measurable.
